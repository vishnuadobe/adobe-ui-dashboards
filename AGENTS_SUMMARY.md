# AGENTS.md - Project Guide Summary

## Overview
This is an **Adobe Experience Manager (AEM) Edge Delivery Services** project. It's a fast, easy-to-author, maintainable web experience built with vanilla JavaScript, CSS3, and semantic HTML5.

**Key Foundation**: Based on https://github.com/adobe/aem-boilerplate/

---

## Key Technologies
- **Edge Delivery Services for AEM Sites** - https://www.aem.live/
- **Vanilla JavaScript** (ES6+) - no transpiling, no build steps
- **CSS3** - modern features, NO Tailwind or CSS frameworks
- **HTML5** - semantic markup generated by aem.live backend
- **Node.js** - tooling support

---

## Setup & Development

### Installation & Running
```bash
npm install                                    # Install dependencies
npm run lint                                   # Run linting
npm run lint:fix                               # Auto-fix lint issues

# Start dev server (background recommended)
npx -y @adobe/aem-cli up --no-open --forward-browser-logs
# Or if AEM CLI installed globally:
aem up
```

**Dev Server**: `http://localhost:3000` with auto-reload

### Testing Content
- Create static HTML files in `drafts/` folder (`.html` or `.plain.html`)
- Start dev server with: `aem up --html-folder drafts`
- Inspect page content with:
  - `curl http://localhost:3000/path/to/page`
  - `curl http://localhost:3000/path/to/page.md`
  - `curl http://localhost:3000/path/to/page.plain.html`

## Code Style Guidelines

### JavaScript
✅ **DO:**
- Use ES6+ features (arrow functions, destructuring, template literals)
- Follow **Airbnb ESLint** rules (pre-configured)
- Always include `.js` file extensions in imports
- Use Unix line endings (LF)

### CSS
✅ **DO:**
- Follow **Stylelint** standard configuration
- Use modern CSS: Grid, Flexbox, Custom Properties
- **Mobile-first** responsive design
- Media queries: `min-width` at 600px, 900px, 1200px
- **Scope all selectors to the block** (important!)

❌ **DON'T:**
- Use `.item-list` → ✅ Use `.{blockname} .item-list`
- Use `.{blockname}-container` or `.{blockname}-wrapper` (reserved for sections)

### HTML
✅ **DO:**
- Use semantic HTML5 elements
- Include ARIA labels & proper heading hierarchy
- Follow AEM markup conventions for blocks & sections

## Core Concepts

### 1. Content Structure
- Pages are divided into **sections**
- Sections contain **default content** + **blocks**
- Authors create content through CMS

### 2. Blocks
**Purpose**: Reusable components that add styling & functionality

**Block Structure**:
```javascript
/**
 * loads and decorates the block
 * @param {Element} block The block element
 */
export default async function decorate(block) {
  // 1. Load dependencies
  // 2. Extract configuration
  // 3. Transform DOM
  // 4. Add event listeners
}
```

**Best Practices**:
- Define initial HTML structure (contract between author & developer)
- Use `curl` & `console.log` to inspect delivered HTML
- Handle gracefully when authors omit/add fields
- Self-contained & reusable
- Responsive & accessible by default
- Naming: `blockname.css`, `blockname.js`

### 3. Auto-Blocking
- Create blocks based on content patterns (not explicitly authored)
- See `buildAutoBlocks` function in `scripts.js`

### 4. Three-Phase Page Loading

| Phase | Purpose | Includes |
|-------|---------|----------|
| **Eager** | Get to LCP | Page structure, sections, first section |
| **Lazy** | Load all content | Header, footer, remaining sections |
| **Delayed** | Non-critical | Martech, analytics (acceptable performance penalty) |

## Deployment & Publishing

### Environment URLs
- **Local Dev**: `http://localhost:3000`
- **Production Preview**: `https://main--adobe-ui-dashboards--vishnuadobe.aem.page/`
- **Production Live**: `https://main--adobe-ui-dashboards--vishnuadobe.aem.live/`
- **Feature Preview**: `https://{branch}--adobe-ui-dashboards--vishnuadobe.aem.page/`

### Publishing Workflow

1. **Push to feature branch**
2. **AEM Code Sync** auto-processes → feature preview available
3. **Test & Optimize**
   - Check PageSpeed Insights at feature preview URL
   - Fix issues → target score: 100
4. **Open Pull Request** to `main`
   - Include link to feature preview showing your changes
   - ⚠️ **PR WILL BE REJECTED without preview link**
   - If no existing page: create static HTML file in `drafts/`
5. **Verify Status**: `gh pr checks`
6. **Review & Merge** by human reviewer
7. **Production Update**: AEM Code Sync updates `main` → production

## Blocks Reference

### Cards Block (`blocks/cards/`)

**Purpose**: Display app tiles with drawer details panel

**Features**:
- Responsive grid layout (1 column mobile, 2 columns tablet, 3 columns desktop)
- App icon/monogram display
- Drawer sidebar with app details, image, and launch button
- Optional app preview images
- Hover animations on buttons
- Keyboard accessible (Escape to close drawer)

**Authoring Structure** (5-column table):

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| Icon | Image/Icon | Yes | App icon or SVG |
| Title | Text | Yes | App name |
| Description | Text | Yes | App description (shown in drawer) |
| Link | URL | Yes | Link to app or href attribute |
| Image | Image | No | App preview/screenshot (displays in drawer sidebar) |

**Example**:
```
| Icon | Title | Description | Link | Image |
| [icon.svg] | Adobe Forum | Collaborative forum... | https://example.com | [screenshot.png] |
```

**Styling Notes**:
- All selectors scoped to `.app-cards-*` namespace
- Mobile-first responsive design
- Uses Spectrum design tokens
- Drawer opens from right side on desktop, full-screen on mobile
